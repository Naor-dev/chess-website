# Design Topics Reference

This document provides a quick reference for the 12 design topic categories used in the web design knowledge base.

## Topic Categories

### 1. Color Theory
Color palettes, contrast, harmony, color psychology, accessibility (WCAG), and color scheme selection.

**Use when**: Choosing colors, creating palettes, ensuring contrast ratios, understanding color psychology.

### 2. Typography
Font selection, pairing, hierarchy, readability, web fonts, type scales, and typographic systems.

**Use when**: Selecting fonts, establishing type hierarchy, improving readability, pairing typefaces.

### 3. Layout
Grid systems, spacing, composition, visual flow, alignment, flexbox, CSS grid, and whitespace.

**Use when**: Structuring pages, creating grids, managing spacing, establishing visual flow.

### 4. UX Principles
User experience fundamentals, usability heuristics, accessibility, user research, and cognitive load.

**Use when**: Improving usability, conducting user research, ensuring accessibility, reducing cognitive load.

### 5. UI Patterns
Common UI components (buttons, modals, cards), interaction patterns, and design patterns.

**Use when**: Implementing standard components, following established patterns, creating consistent interfaces.

### 6. Responsive Design
Mobile-first approach, breakpoints, fluid layouts, media queries, and device adaptation.

**Use when**: Building for multiple devices, setting breakpoints, implementing fluid designs.

### 7. CSS Techniques
CSS properties, selectors, animations, transitions, modern CSS features, and implementation.

**Use when**: Writing CSS, implementing animations, using modern CSS features.

### 8. Visual Hierarchy
Emphasis, contrast, scale, visual weight, attention flow, and focal points.

**Use when**: Establishing importance, guiding attention, creating visual emphasis.

### 9. Branding
Brand identity, consistency, style guides, design systems, and visual identity.

**Use when**: Creating brand guidelines, ensuring consistency, building design systems.

### 10. Navigation
Menus, wayfinding, information architecture, site structure, and navigation patterns.

**Use when**: Designing navigation, structuring sites, improving wayfinding.

### 11. Forms & Inputs
Form design, input validation, error handling, user input patterns, and form UX.

**Use when**: Designing forms, handling validation, improving form usability.

### 12. Performance
Load times, optimization, perceived speed, Core Web Vitals, and performance best practices.

**Use when**: Optimizing performance, improving load times, addressing Core Web Vitals.
